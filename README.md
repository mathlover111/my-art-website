# my-art-website
## 我的藝術網站：開發歷程

第一個版本 (v1.0)
這個版本雖然簡約，但完整呈現了動態網頁的核心邏輯：
* **資料持久化**：使用 `localStorage` 解決網頁重整後資料流失的問題，並透過 `JSON.parse` 與 `JSON.stringify` 處理陣列資料。
* **動態 DOM 操作**：透過 `createElement` 與 `appendChild` 讓留言即時顯示。
* **介面配置**：利用 CSS `position: fixed` 優化留言列表的顯示位置。
##  Debug 與失敗經驗紀錄
在開發過程中，我經歷了多次邏輯錯誤，以下是 20 次修改中最重要的幾個修正點：
1. **重複渲染問題**：修正前留言會無限重複出現。解決方案是在 `showComments()` 函式開頭加入 `list.innerHTML = ""` 清空舊內容。
2. **空字串攔截**：加入 `if (text === "") return;` 邏輯，防止無效的留言內容送出。
3. **字型與排版校正**：針對不同瀏覽器對標點符號（如逗號）的渲染差異，微調字型設定與行高，確保閱讀體驗。

##  第二個版本 (v2.0)：Firebase 雲端同步版
這是專案目前的最新狀態，實現了真正的全球訪客互動。

* **核心思維**：意識到本地儲存的侷限性，主動尋求「後端解決方案」。
* **技術突破**：
    * **雲端資料庫串接**：整合 Firebase Realtime Database，達成跨裝置資料同步。
    * **即時監聽機制**：運用 `on("child_added")` 技術，讓新留言不需重新整理即可自動跳出。
    * **架構決策**：從單機儲存轉向 API 導向的開發模式。

## 第三個版本 (v3.0)：進階管理與身份驗證版
這是本專案目前最完整的版本，重點在於實現了**「管理員與一般使用者的權限區分」**。

核心思維：
網頁不應只是開放留言，還需要有管理機制（刪除不當留言）以及身分識別（誰留了什麼）。

技術突破（重點標記）：

Google OAuth 整合：導入 firebase-auth.js，實作第三方登入機制。

權限控制邏輯 (Admin Privilege)：

定義硬編碼的 adminID，實作「僅限管理員可見」的圖片上傳面板 (adminPanel)。

實作留言保護機制：只有留言者本人或管理員可以看見並執行「刪除」功能。

動態內容管理：將原本寫死的 <img> 改成由資料庫讀取的 loadImages() 函式，達成「免改代碼即可更新畫作」的目標。

時間戳記處理：學會使用 Date.now() 與格式化輸出，讓留言顯示正確的發布時間。
